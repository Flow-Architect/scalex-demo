# ScaleX Demo Script

Target length: three minutes.

## Core Message

Enterprise teams want AI agents to help run client operations, but they cannot let raw agents touch
money, vendors, client workflows, or internal systems without proof, policy, money control, and
audit.

ScaleX is the governed execution layer for revenue-backed client operations. Hermes plans the
work. Stripe proves finance state. NemoClaw / NeMo / local policy checks risk. ScaleX blocks
unsafe execution, records evidence, and reports protected profit.

## Recording Flow

### 0:00-0:10 - Problem

Show the dashboard hero and active Northstar operation.

Voiceover:

> Enterprises want AI agents, but they cannot let raw agents touch money, vendors, client
> workflows, or systems without proof, policy, money control, and audit.

### 0:10-0:25 - Product

Show the ScaleX headline, active operation card, and control-room layout.

Voiceover:

> This is ScaleX: governed execution for revenue-backed client operations.

### 0:25-0:45 - Operation Economics

Show Northstar Dental Group / Client Implementation Launch:

- revenue secured: `$8,500`
- approved delivery cost basis: `$3,935`
- blocked risk: `$3,200`
- protected profit: `$4,565`
- protected margin: `53.7%`
- margin floor: `50.0%`

Voiceover:

> ScaleX knows the client, the money, the approved cost basis, the risky spend to block, and the
> protected profit before the operation moves forward.

### 0:45-1:10 - Control Stack

Show the connected stack:

- Hermes planning
- Stripe finance proof
- NemoClaw / NeMo / local policy guardrails
- ScaleX control plane

Voiceover:

> Hermes creates the plan. Stripe provides sandbox/test finance proof. NemoClaw, NeMo, or local
> policy checks risky actions. ScaleX executes only what is allowed and records the proof.

### 1:10-1:45 - Start Governed Run

Click `Start Governed Run`.

Show the rails:

- input passed
- Hermes plan created
- planning approved
- Stripe finance proof verified
- revenue gate passed
- guardrails checked
- controlled setup spend approved
- risky vendor action blocked
- evidence recorded
- output honesty verified
- profit outcome protected

Pause on the blocked-risk rail.

Voiceover:

> The important moment is not just execution. ScaleX blocks the unsafe vendor action before it can
> create uncontrolled spend exposure.

### 1:45-2:15 - Evidence Ledger

Open Evidence Ledger.

Show:

- planning proof
- finance proof
- policy checks
- approved spend
- blocked risk
- labor-cost proof
- paid-state honesty
- no live-money mode
- no secrets stored
- final profit outcome

Voiceover:

> Every meaningful decision is recorded: what happened, who or what acted, what was approved, what
> was blocked, and what proof exists.

### 2:15-2:45 - Profit Outcome

Show the profit report.

Voiceover:

> The final outcome is business-readable: revenue, approved delivery costs, blocked risk,
> protected profit, protected margin, and the margin floor.

### 2:45-3:00 - Close

Return to the dashboard or control stack.

Voiceover:

> ScaleX gives enterprises a safe way to let AI execute paid client operations without losing
> control of money, margin, policy, or audit.

## Truthfulness Notes

- Northstar Dental Group is synthetic.
- No patient data, PHI, SSNs, tax IDs, bank data, payroll records, or real customer workflows are
  used.
- Judge Demo Mode works without secrets.
- Stripe live-money execution is not implemented.
- Verified Live Mode is future-only and locked.
- Telegram approval and MCP server access are not implemented.
- Real Hermes, NemoClaw, NeMo Guardrails, or Stripe usage should be claimed only when runtime
  evidence proves the configured path.
